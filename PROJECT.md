# 프로젝트 기준 문서 (PROJECT.md) — menueye_v2

> 이 문서는 **v2(한국어 메뉴판 버전)의 단일 기준(Source of Truth)** 이다.
> 모든 코드·노트북·문서는 이 문서에 맞춰 작성하고, 충돌 시 이 문서를 따른다.
> (유럽권 v1 폴더 `MenuEye_프로젝트`는 그대로 보존하며 이 폴더에서 변경하지 않는다.)

---

## 방향 전환 (v1 → v2)
- **v1**: 다국어(유럽 라틴) 메뉴판 → OCR → **한국어**로 해석. 병목 = 유럽어 OCR 정확도(특수문자·장식체·저대비).
- **v2**: **한국어(한식) 메뉴판** → OCR → **다국어(영어 우선)**로 해석. 병목이던 OCR이 쉬워지고(한국어 OCR 성숙), 모국어라 정답 검증이 쉽다.
- 문제의식도 재정의: **방한 외국인 관광객이 한식 메뉴를 못 읽고 못 이해하는 문제**를 CV+LLM으로 해결(K-관광).

## 주제
OCR 기반 한식 메뉴판 인식 및 다국어 LLM 푸드 가이드

## 한 줄 정의
한식 메뉴판 사진 한 장을 찍으면, 글자를 인식(CV/OCR)하고 LLM이 메뉴를 **다국어로 해석·안내**해 주는 서비스. (CV + LLM 결합 응용)

## 핵심 흐름 (이게 프로젝트의 전부)
1. **이미지 촬영** — 한식 메뉴판 사진 한 장
2. **인식 (CV/OCR)** — PaddleOCR(한국어)로 글자 검출·인식
3. **해석·가이드 (LLM)** — 인식된 메뉴에 대해 (목표 언어 = 영어 등 다국어):
   - 전체 메뉴 소개
   - 맛의 특징 간단히 (맵기, 향신료 등)
   - 식재료 소개
   - 알레르기 주의 안내

## OCR 전략 (핵심)
- **주력 = PaddleOCR(한국어 모델).** 이걸로 **베이스라인 → 개선(전처리 등) → 오류사례 분석**의 모델링 스토리를 만든다.
- **고성능 비교 기준 = CLOVA OCR(네이버).** 상용 고정확 OCR을 벤치마크로 두고 PaddleOCR 결과와 비교해, 개선 여지·한계를 정량적으로 보여준다.
- 전처리(CLAHE 등)·채점 도구(compare_menu.py) 등 v1에서 검증된 방법론은 언어만 바꿔 재사용한다.

## 알레르기 안내 규칙 (중요, v1과 동일)
- **단정하지 않는다.** "이건 알레르기를 유발한다" 같은 표현 금지.
- 가게마다 재료가 다르므로, 반응 가능성이 있는 **'의심 재료'만 간단히** 짚는다.
- 마지막은 항상 **"정확한 건 식당에 직접 문의"** 방향으로 유도한다.
- 이 규칙은 LLM 프롬프트에 명시적으로 포함한다.

## 범위 밖 (하지 않는 것, v1과 동일)
- **사용자 조건 기반 '추천 / 필터링' 기능 없음.** 본질은 '해석·가이드(이해시키기)'.
- **헬스케어 주제 아님.**
- **가격은 결과에 포함하지 않는다.** (항목 구분·레이아웃 단서로만 사용)

## 기술 스택
- **CV/OCR**: PaddleOCR(한국어, 검출 DBNet + 인식 SVTR/CRNN) — 주력 / CLOVA OCR — 비교 기준
- **LLM**: API 기반 다국어 해석·안내 생성 (OpenAI gpt-4o 등)
- **산출물**: 구현 코드(.ipynb) + 채점·오류분석 자료

## 대상 언어
- **입력(메뉴판) = 한국어(한식).**
- **출력(해석) = 다국어, 영어 우선** (이후 일본어·중국어 등 확장 가능).

## 데이터
- **크롤링으로 수집한 한식 메뉴판 이미지** (가장 중요한 도메인 데이터).
- ※ 유럽권 ICDAR 2019 MLT 이미지·GT는 v2에서 **사용하지 않는다**(유럽 메뉴 전용). ICDAR에 한국어 구간이 있으나 그것은 **거리 간판**이지 메뉴판이 아니라 부적합.
- 정답(truth)은 한국어 원어민이 직접 작성 → 인식 결과 검증이 쉬움(v1의 raifort→roquefort 같은 오인식 놓침 위험 해소).

## 모델 작업 흐름 (해커톤 채점 정렬)
1. **베이스라인** — PaddleOCR(한국어) 그대로 채점 (정확도·CER 측정)
2. **개선** — 전처리(CLAHE 등) 또는 파인튜닝
3. **재측정** — 개선 전/후 비교, 그리고 **CLOVA OCR과 비교**
4. **오류사례 분석** — 틀린 케이스 유형별 정리 (레이아웃·저대비 등)
5. **결과 정리** — 표·그래프

## 재사용 자산 (v1에서 가져옴)
- `scripts/compare_menu.py` — 정답(truth) vs 출력(JSON) 요리 단위 대조 채점(일치·누락·오인식·병합·허위). 언어 무관.
- `scripts/menu_to_prompt.py` — 이미지→OCR→LLM 해석 변환 파이프라인. **한국어 지원 적응 필요**(WORKLOG 참고).
- `scripts/run_ocr.py` — PaddleOCR 래퍼(lang 지정 가능).
- `notebooks/` — EDA 뼈대, PaddleOCR 파인튜닝 노트북.
- 기타 스크립트(compare_ocr·tess_*·make_label_from_icdar 등)는 v1 참고용으로 함께 복사됨(대부분 유럽/ICDAR 전용).
